## S-BERT 임계값 최적화 실험 상세 설명

---

### 실험이 해결하려는 문제

현재 시스템은 두 과목의 S-BERT 유사도가 **0.7 이상**이면 "유사 과목"으로 판정합니다. 그런데 이 0.7이라는 숫자는 어떤 근거로 정한 걸까요? 이 실험은 **"0.7이 정말 최적인가, 아니면 다른 값이 더 좋은가"** 를 데이터로 검증합니다.

---

### 핵심 문제: 정답 레이블이 없다

Precision/Recall을 계산하려면 **"이 과목 쌍은 실제로 유사하다"** 는 정답 레이블이 필요합니다. 하지만 수백 개 과목의 모든 쌍에 사람이 직접 레이블을 붙이는 건 비현실적입니다.

그래서 **의사 정답(Pseudo Ground Truth)** 을 만듭니다.

---

### 의사 정답 설계: TF-IDF를 심판으로 사용

```
TF-IDF 유사도(과목 A, 과목 B) >= 0.30  →  "실제 유사 쌍" (GT = 1)
TF-IDF 유사도(과목 A, 과목 B)  < 0.30  →  "비유사 쌍"   (GT = 0)
```

**왜 TF-IDF를 심판으로 쓰는가:**

| 비교 항목 | S-BERT | TF-IDF |
| --- | --- | --- |
| 측정 방식 | 문장 전체의 **의미적** 유사도 | **공통 키워드** 빈도 기반 |
| 문제점 | "학습한다", "이해하고" 같은 상투어에도 높은 유사도 부여 가능 | 내용 없이 상투어만 있으면 낮은 유사도 |
| 서로 독립적? | ✅ 완전히 다른 방법론 | ✅ |

TF-IDF가 높다는 것은 두 과목의 개요에 **같은 전문 용어가 많이 등장**한다는 의미이므로, 사람이 읽어도 "비슷한 과목"이라고 느낄 가능성이 높습니다. 즉 TF-IDF는 **설명 가능한(explainable) 기준**입니다.

---

### 전처리: 어떤 쌍을 실험 대상에서 제외하는가

```
전체 과목 N개 → 가능한 쌍: N(N-1)/2 개
      ↓
❌ 같은 학수번호 쌍 제외  (동일 과목이 학과만 다른 것)
❌ 연계 과목 쌍 제외       (미분적분학1 ↔ 미분적분학2 등)
      ↓
✅ 유효 쌍만으로 실험
```

이건 기존 course_similarity_graph.py의 production 로직을 그대로 재사용한 것입니다. 연계 과목은 당연히 유사도가 높게 나오므로 포함시키면 결과가 왜곡됩니다.

---

### 실험의 흐름

```
threshold = 0.60 일 때:

  [전체 유효 쌍]
  ┌──────────────────────────────────────────┐
  │  TF-IDF GT=1 (실제 유사)  │  GT=0 (비유사) │
  ├────────────────┬──────────┼───────────────┤
  │  S-BERT >= 0.6 │    TP    │      FP       │  ← SBERT가 "유사"라고 함
  │  S-BERT <  0.6 │    FN    │      TN       │  ← SBERT가 "비유사"라고 함
  └────────────────┴──────────┴───────────────┘

  Precision = TP / (TP + FP)   → SBERT가 유사라고 한 것 중 진짜 유사한 비율
  Recall    = TP / (TP + FN)   → 진짜 유사한 쌍 중 SBERT가 잡아낸 비율
  F1        = 조화평균(P, R)
```

threshold를 0.60 → 0.65 → 0.70 → ... → 0.90 으로 올리면:

- **Threshold ↑** → SBERT가 "유사"라고 판정하는 기준이 까다로워짐
    - FP 감소 (비유사를 유사라고 잘못 잡는 것 줄어듦) → **Precision ↑**
    - FN 증가 (실제 유사인데 못 잡는 것 늘어남) → **Recall ↓**
    - 엣지 수(그래프 연결선) 감소

---

### 출력 그래프 4종의 의미

**① `threshold_curves.png`**

- 왼쪽: threshold가 변할 때 P/R/F1 선이 어떻게 움직이는지
- 오른쪽: threshold별로 실제 그래프에 생기는 엣지(연결선) 수
- → *"F1 최고점이 어디인가"*, *"현재 0.7에서 엣지를 몇 개 더 허용/제한해도 되는가"* 파악

**② `pr_curve.png`** (Precision-Recall Curve)

- X축=Recall, Y축=Precision, 각 점이 하나의 threshold
- 오른쪽 위(P=1, R=1)에 가까울수록 좋음
- 점선(무작위 분류기 기준선) 대비 얼마나 위에 있는지 확인
- → *"S-BERT가 TF-IDF 기준과 얼마나 일치하는가"* 전체적인 판별력 평가

**③ `tp_fp_fn_stacked.png`**

- threshold별로 TP(올바르게 잡은 쌍) vs FP(잘못 잡은 쌍) 분해
- 위에 주석으로 FN(놓친 쌍) 수 표시
- → *"이 threshold에서 어떤 종류의 오류가 더 비싼가"* 결정 시 사용

**④ `sbert_score_distribution.png`**

- GT=1(실제 유사) 쌍과 GT=0(비유사) 쌍의 SBERT 점수 분포를 겹쳐서 표시
- → **두 분포가 잘 분리되어 있으면**: threshold 설정만으로 효과적 분류 가능
- → **두 분포가 많이 겹치면**: 아무 threshold를 써도 한계가 있음 → 모델 자체를 바꿔야 한다는 신호

---

### 실험 결과를 어떻게 해석하나
            
```
예상 시나리오:

threshold=0.65  P=0.72  R=0.81  F1=0.76  엣지=2,100
threshold=0.70  P=0.79  R=0.68  F1=0.73  엣지=1,400   ← 현재 기준
threshold=0.75  P=0.85  R=0.55  F1=0.67  엣지=  900
```

- F1 최고가 0.65이면 → "현재 0.70은 너무 엄격, 0.65로 낮춰야"
- F1 최고가 0.75이면 → "현재 0.70은 너무 관대, 0.75로 올려야"
- F1이 전 구간에서 비슷하면 → threshold보다 모델/전처리가 더 중요

**그래프 구축 관점에서의 트레이드오프:**

| 목적 | 권장 threshold |
| --- | --- |
| 추천 정확도 중시 (노이즈 없는 연결) | F1 최고점보다 오른쪽 (높은 값) |
| 커버리지 중시 (관련 과목 최대한 연결) | F1 최고점보다 왼쪽 (낮은 값) |
| 균형 | F1 최고점 |
